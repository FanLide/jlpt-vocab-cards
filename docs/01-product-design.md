# 产品设计文档（PRD）- JLPT 万词对策单词卡片

## 1. 背景与问题
- 纸质书《新日语能力考试 万词对策》有**单词 + 例句**，但通勤场景下“听到记不住→翻书”成本高。
- 目标是在电车上用手机快速复习：**课程→音频→卡片**强绑定，支持离线。

## 2. 目标（MVP）
- Web（React）实现：
  1) 书籍结构浏览（章节/课程）
  2) 课程内音频播放
  3) 课程内卡片复习（全屏模式 / 列表模式）
  4) 离线：播放即缓存 + 本课预缓存（下载）
  5) 记忆辅助：默认遮挡，长按揭示（松开隐藏）

## 3. 非目标（MVP 不做）
- 音频与卡片的时间轴自动对齐/自动切换
- 完整 SRS（间隔重复算法）——后续迭代

## 4. 用户故事（MVP）
1) 我在课程列表点击“第 X 课”，进入课程页。
2) 我点击播放音频，能继续播放/暂停/拖动进度；下次离线也能播。
3) 我在全屏卡片复习：默认只看到“编号 + 平假名”，长按 1 秒看到详情，松开隐藏。
4) 我切换到列表模式，快速定位某个卡片并进入全屏复习。
5) 我点“下载本课”，把音频预缓存，地铁无网也能用。

## 5. 信息架构
- 书籍（N2 / N3 / 后续等级）
  - 章节
    - 课程
      - 音频
      - 卡片集合

## 6. 关键交互规范
### 6.1 卡片遮挡/揭示
- 默认态：显示
  - 编号（index）
  - reading（平假名）
- 长按（press >= 1000ms）：进入“揭示态”，显示
  - word（日语原文）
  - meaning.zh（中文）
  - meaning.en（可选）
  - sentence.ja（日语例句）
  - sentence.zh（可选）
- 松开：立即回到默认态
- 可选增强：提供“锁定显示”按钮（方便查词/核对）

### 6.2 音频缓存
- 播放即缓存：用户播放时后台缓存该课音频
- 预缓存按钮：用户点击“下载本课”后，显示进度与完成状态
- 提供入口：缓存占用、清理缓存

## 7. 数据字段（卡片）
- id（稳定唯一）
- index（课程内序号）
- word（如：影響）
- reading（如：えいきょう）
- meaning: { zh: string, en?: string }
- sentence: { ja: string, zh?: string }
- tags?: string[]（可选）

## 8. 质量标准（验收）
- 离线打开已访问课程：卡片可正常浏览
- 已缓存音频：断网情况下可播放
- 卡片长按揭示在移动端（iOS Safari/Chrome）稳定可用
- 课程切换不会串课（音频与卡片对应一致）
