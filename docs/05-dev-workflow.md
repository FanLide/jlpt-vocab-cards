# 标准流水线工作模式（赛博妲己 × OpenCode × Claude）

> 目标：更快迭代、更少返工、问题分级清晰、交付可验收。

## 角色分工
- **赛博妲己（总管）**：需求拆解、优先级、风险控制、验收标准、合并决策与最终交付。
- **OpenCode（开发）**：实现功能/修复、提交代码（commit）、补充脚本/测试。
- **Claude（Review / QA）**：代码审查 + smoke test（lint/build/关键交互），输出问题清单（按严重度排序）。

## 一轮迭代的固定产出
1) **OpenCode 输出**
- 代码变更 + 至少 1 个 commit
- 简短说明：改了什么、怎么验收

2) **Claude 输出**
- 问题清单（按严重度：🔴高 / 🟠中 / 🟡低）
- 已修复项（如有）
- 风险与建议（可操作）

3) **赛博妲己输出（对你交付）**
- 本轮结论（1 句话）
- 变更摘要（最多 7 条）
- 需要你确认的外部操作（如部署/配置变更）
- 验收标准

## 退出条件（Definition of Done）
- Claude review 只剩 **低风险/纯 UX** 问题，且
- `pnpm lint` + `pnpm build` 通过
- 主链路可用：目录 → 课程 → 音频 → 卡片（长按/滑动/搜索/跳转）

## 执行节奏建议
- 优先处理 🔴/🟠，🟡 统一放到最后一轮收尾
- 每轮变更保持“小步可回滚”，避免大爆炸式提交

## 常用命令
- 启动开发：`cd web && pnpm dev`
- 质量检查：`cd web && pnpm lint && pnpm build`
- 导入文本：`node scripts/import-text-to-json.mjs`

## tmux 约定
- 每个项目一个 tmux session，命名使用项目名（避免混淆）。
